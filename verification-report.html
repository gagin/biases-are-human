How IBI is actually computed

IBI is not a simple directional count (A>B vs B>A). It's the mean signed difference normalized by each pair's scale range. From src/bias_bench/scoring.py:143-228:

For each pair: diff = value_A - value_B normalized_diff = diff / (max_choice_value - min_choice_value) IBI = mean(normalized_diffs) across all 30 pairs

This means a pair where A=9, B=1 on a 1-10 scale (normalized diff = 8/9 = 0.889) contributes much more than a pair where A=230, B=320 on a 20-900 scale (normalized diff = -90/880 = -0.098).

Layer 6: Adversarial Samples

6a. The anc- vs rel- split — CRITICAL

The magnitude implicit items come in two subtypes: anc- (anchoring, 15 pairs) and rel- (relevance, 15 pairs). For GPT-5.4:

  • anc- pairs: 14 ties, 0 A>B, 1 B>A. Almost no bias signal.
  • rel- pairs: 15 A>B, 0 ties. Maximum possible bias signal.

What the rel- items actually test: They present the same numerical difference in two framings:

Version A — unresolved prospective problem
"A research lab is applying for a grant. The minimum required budget is $2.85M. The lab has secured $2.62M. The shortfall is $230K..." → Rate the significance of this difference (1-10)
Version B — already resolved retrospective
"A research lab completed a study. The projected budget was $2.85M; actual was $2.62M — $230K under budget. The lab has already published results..." → Rate the significance of this difference (1-10)

Every rel- pair follows this pattern: version A = threshold still at stake, version B = already resolved. The numbers are identical; the temporal framing differs.

THIS IS NOT ANCHORING. This is a relevance/stakes framing effect: the same gap is rated as more significant when it represents a live problem than a resolved historical fact. This is arguably a correct response — a $230K shortfall matters more when you still need the money than when the project is already done.

Impact on reported results: GPT-5.4's "magnitude IBI" of 0.371 is almost entirely driven by rel- items (contributing ~0.37) with anc- items contributing ~-0.003. The paper and substack describe this as "anchoring bias" but for GPT-5.4, it's really a relevance-framing effect. Other models may show different anc/rel splits.

This may not invalidate the finding but it demands different interpretation. The rel- items may be measuring something real and interesting — context-dependent significance assessment — but calling it "anchoring" is potentially misleading. This should be investigated across all models.

6b. Arena correlation model selection

As discussed above: ρ=1.000 holds for the 6 base models. Including DeepSeek R1 and Gemini 3 Flash Preview drops to ρ=0.833 (still p=0.01). The perfect correlation depends on the selection.

6c. Duplicate scores in DB

The data-gather agent noted that each score row appears 4x in the DB. This suggests the scoring was run multiple times without deduplication. The values are identical across duplicates, so it doesn't affect correctness — but it's a data hygiene issue.

6d. Grok reasoning raw response

Grok's reasoning mode returns <reasoning>...</reasoning> tags before the answer letter. The parser correctly extracts the final letter after the tags. In the anc-005 trace above, Grok's reasoning explicitly identifies the preamble as "irrelevant, possibly a distractor" — and then gives the same answer anyway. This is the "knows it's wrong, does it anyway" pattern at the individual response level.
